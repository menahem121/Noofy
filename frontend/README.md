# Noofy Frontend

React and TypeScript frontend for the Noofy desktop app.

## Structure

- `src/features/home/`: the real Home Page screen and its tests.
- `src/lib/api/`: small backend API client. Frontend code calls Noofy `/api` endpoints only.
- `src/styles/`: app-wide design tokens and layout styles.

## Development

```bash
npm install
npm run dev
```

The dev server runs at `http://127.0.0.1:5173/` and proxies same-origin `/api` requests to the backend at `http://127.0.0.1:8765` from the dev-server process.

Keep browser development on the same-origin `/api` proxy by default. This is important when Noofy runs on a remote GPU server and the UI is opened through a browser or forwarded port: a frontend URL like `http://127.0.0.1:<backend-port>/api` points at the browser machine, not necessarily the Noofy server. Override the API base URL with `VITE_NOOFY_API_BASE_URL` only when the desktop shell or an explicit remote setup provides a correct reachable backend URL.

## Desktop Shell

Run the Tauri shell from this directory:

```bash
npm run tauri:dev
```

The shell starts the Python backend with a free localhost port and injects runtime config before React renders:

```ts
window.__NOOFY_RUNTIME_CONFIG__ = {
  apiBaseUrl: "http://127.0.0.1:<port>/api",
  apiToken: "<per-launch-token>"
}
```

In development, the shell can use the source-checkout backend venv. Packaged
builds are stricter: they use bundled Noofy runtime resources and do not fall
back to system Python, Homebrew Python, Conda, or `backend/.venv`. `tauri build`
also verifies that the packaged Python and `uv` runtime artifacts are present
before creating an installer.

Use `NOOFY_BACKEND_PYTHON` and `NOOFY_BACKEND_DIR` only for source-checkout
development. Product installers run ComfyUI in managed mode and point the
backend at packaged ComfyUI/runtime resources.

`VITE_NOOFY_API_TOKEN` is available for token-protected browser testing, but desktop builds should use the runtime config value generated by Tauri.
